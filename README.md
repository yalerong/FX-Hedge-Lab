# FX Hedge Lab 使用手册

FX Hedge Lab 是一个本地运行的外汇风险与锁汇模拟工作台。你可以用它录入未来外币收付款，查看净敞口，生成锁汇建议，记录锁汇操作，并在到期后做收益回测。

这个工具适合演示、试算和内部流程讨论。它默认把数据保存在你电脑本地，不需要数据库，也不需要部署服务器。

## 你可以用它做什么

- 录入未来要收或要付的外币金额。
- 自动按月份和币种汇总净敞口。
- 根据企业类型和套保比例生成锁汇建议。
- 一键把建议填入锁汇单。
- 记录实际执行的锁汇交易。
- 查看中性、乐观、悲观、自定义汇率场景下的预计损益。
- 到期后录入实际汇率，回测锁汇交易贡献。
- 每天自动拉取一次汇率并缓存到本地。

## 页面区域说明

### 1. 汇率与缓存

这里显示当前汇率来源、汇率状态和更新时间。

右上角有两个按钮：

- `刷新汇率`：立即从汇率接口重新拉取一次数据。
- `恢复样例`：把本地数据恢复成系统内置的演示样例。

默认汇率接口：

```text
https://open.er-api.com/v6/latest/USD
```

汇率会缓存到本地，默认 24 小时更新一次。

### 2. 添加敞口

敞口就是未来还没锁定汇率的外币收付款。

字段说明：

- `到期日`：预计收款或付款发生的日期。
- `币种`：例如 `USD`、`EUR`、`JPY`。
- `方向`：
  - `未来收外币`：例如出口收美元。
  - `未来付外币`：例如进口付欧元。
- `金额`：外币金额。
- `概率`：这笔业务发生的概率，默认 `1`，表示 100% 会发生。
- `风险类型`：
  - `资产负债表套保`
  - `现金流套保`
  - `合同/订单套保`
- `备注`：写给自己看的说明。

填完后点击：

```text
保存敞口
```

保存成功后，页面顶部会显示保存提示，并自动跳到“敞口明细”。

### 3. 执行锁汇

这里用来记录你实际做过的锁汇交易。

字段说明：

- `交易日`：锁汇交易发生日期。
- `到期日`：锁汇交割日期。
- `币种`：锁汇币种。
- `操作`：
  - `卖出外币/远期结汇`：适合未来要收外币的情况。
  - `买入外币/远期购汇`：适合未来要付外币的情况。
- `锁汇金额`：锁定的外币金额。
- `锁定汇率`：实际成交汇率。
- `备注`：例如银行、交易编号、业务说明。

可以手动填写，也可以在“锁汇建议”里点击：

```text
按建议填入锁汇单
```

系统会自动把建议金额、方向、币种和汇率填进表单，你确认后再点击“记录锁汇”。

### 4. 锁汇建议

这里展示系统根据当前净敞口生成的建议。

你会看到：

- 推荐锁汇币种。
- 推荐锁汇金额。
- 操作方向。
- 当前参考汇率。
- 目标套保比例。
- 预计影响的损益科目。

建议金额的基本思路是：

```text
建议锁汇金额 = 业务敞口 × 目标套保比例 - 已经锁掉的金额
```

如果已经锁掉的金额足够多，系统就不会继续建议重复锁汇。

### 5. 净敞口

这里是汇总表。它把你录入的敞口和已经执行的锁汇交易放在一起看。

主要字段：

- `业务敞口`：业务本身带来的外币风险。
- `已锁敞口`：已经通过锁汇交易抵掉的部分。
- `剩余敞口`：还暴露在汇率波动里的部分。
- `当前汇率`：系统当前使用的参考汇率。
- `人民币风险`：剩余敞口按当前汇率折算的大致人民币金额。

### 6. 敞口明细

这里列出所有已经保存的敞口。

如果录错了，可以点击对应记录里的：

```text
删除
```

### 7. 锁汇明细

这里列出所有已经记录的锁汇交易。

如果录错了，也可以删除。

### 8. 预计损益场景

这里用来比较不同汇率场景下的预计损益。

当前支持四类场景：

- `中性`：使用当前汇率。
- `乐观`：按配置里的乐观涨跌幅调整汇率。
- `悲观`：按配置里的悲观涨跌幅调整汇率。
- `自定义`：按配置里的自定义涨跌幅调整汇率。

你会看到这些损益项：

- `未实现汇兑损益`
- `衍生品投资收益`
- `衍生品公允价值变动损益`
- `已实现汇兑损益`
- `合计预计损益`

### 9. 添加到期实际汇率

到期后，如果你知道真实汇率，可以在这里录入：

- `到期日`
- `币种`
- `实际汇率`
- `备注`

保存后，系统会在“回测结果”里计算锁汇贡献。

### 10. 回测结果

回测用来回答一个问题：

```text
这笔锁汇交易到期后，相比实际汇率，贡献了多少人民币损益？
```

它不是预测，而是到期后的复盘。

### 11. 配置

这里可以调整系统行为：

- `汇率接口`：默认使用 ExchangeRate-API 免费接口。
- `缓存小时`：默认 24 小时。
- `风险阈值 CNY`：用于展示风险量级。
- `企业类型`：
  - `综合型`
  - `出口型`
  - `进口型`
- `默认套保比例`：例如 `0.8` 表示建议锁 80%。
- `乐观场景涨跌幅`：例如 `0.03` 表示 +3%。
- `悲观场景涨跌幅`：例如 `-0.03` 表示 -3%。
- `自定义场景涨跌幅`：例如 `0.01` 表示 +1%。

改完后点击：

```text
保存配置
```

## 一个完整示例

假设你有一笔未来美元收款：

1. 在“添加敞口”里填写：
   - 到期日：下个月月底
   - 币种：`USD`
   - 方向：`未来收外币`
   - 金额：`1000000`
   - 概率：`1`
   - 风险类型：`合同/订单套保`
2. 点击“保存敞口”。
3. 查看“锁汇建议”，系统会建议卖出一部分美元。
4. 点击“按建议填入锁汇单”。
5. 检查锁汇金额和锁定汇率。
6. 点击“记录锁汇”。
7. 查看“净敞口”，确认剩余风险降低。
8. 查看“预计损益场景”，比较不同汇率场景。
9. 到期后，在“添加到期实际汇率”录入实际 USD/CNY。
10. 查看“回测结果”。

## 数据保存在哪里

网页录入的数据保存在本地：

```text
data/fx_workspace.json
```

汇率缓存保存在：

```text
data/rates_cache.json
```

这两个文件不会提交到 GitHub。

如果你要清空本地数据，可以直接删除 `data/fx_workspace.json`，然后重新启动服务。也可以在网页右上角点击“恢复样例”。

## 常见问题

### 点击保存没反应怎么办？

先看页面顶部状态条。常见原因：

- 必填项没有填。
- 金额或汇率不是正数。
- 本地服务没有启动。
- 浏览器还缓存着旧脚本。

可以按 `Ctrl + F5` 强制刷新页面。

### 汇率刷新失败怎么办？

系统会继续使用本地缓存。如果本地没有缓存，会使用内置兜底汇率，保证页面能继续运行。

### 为什么没有数据库？

当前版本是本地单机工作台，使用 JSON 文件保存数据，方便演示和快速试算。

正式多人使用时，建议改成：

- SQLite：轻量本地版。
- PostgreSQL：多用户 Web 版。

### 能不能不用网页？

可以。项目还保留了命令行和桌面 GUI。

命令行输出 JSON：

```powershell
python fx_risk_simulator.py sample_data.json --pretty
```

命令行输出解释：

```powershell
python fx_risk_simulator.py sample_data.json --explain
```

桌面 GUI：

```powershell
python fx_risk_gui.py
```
