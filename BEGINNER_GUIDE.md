# 外汇风险模型小白版

参考页面：`https://www.fxexpert.cn/risk.html#1`

页面把外汇风险管理拆成五件事：风险识别、风险测量、风险管理、交易管理、风险回溯。我们的原型也按这个顺序理解。

## 1. 风险识别：先找“外币没锁住”的地方

外汇风险不是先看复杂公式，而是先问一句：

> 未来某个月，公司到底是会多收外币，还是多付外币？

例如：

- 出口订单未来收 USD，是正敞口。
- 进口采购未来付 EUR，是负敞口。
- 银行账户里已有 USD 资产，是正敞口。
- 应付账款里有 USD 负债，是负敞口。

正敞口怕外币贬值，负敞口怕外币升值。

## 2. 风险测量：把汇率变化翻译成人民币损益

核心公式：

```text
汇兑影响 = 外币敞口 x (未来汇率 - 当前基准汇率)
```

如果有 USD 2,000,000 正敞口，当前 USD/CNY 是 7.15，未来变成 7.35：

```text
2,000,000 x (7.35 - 7.15) = 400,000 CNY
```

这表示同样的美元未来能换更多人民币，账面影响是正的。

## 3. 风险管理：比较“套多少”更稳

套保不是预测汇率，而是决定要不要把一部分未来汇率提前锁定。

原型会比较：

- 0% 套保：完全暴露在汇率波动里。
- 25% / 50% / 75% 套保：锁一部分，留一部分。
- 100% 套保：把当前识别出的净敞口全部对冲。

排序标准很朴素：

1. 先看最坏情景下亏得少不少。
2. 再看不同情景之间波动大不大。

## 4. 交易管理：已做过的交易要抵掉敞口

如果公司已经做了远期结汇，例如卖出 USD 700,000 买入 CNY，这部分美元风险已经被锁住。

所以模型会同时输出：

- 未套保敞口：业务本身带来的外币风险。
- 已执行交易后敞口：扣掉已做交易后的剩余风险。

## 5. 风险回溯：检查模型和财务是否对得上

最后一步不是再预测，而是回头检查：

- 交易金额和交易汇率是否匹配。
- 汇率数据是否缺失。
- 模型算出来的损益和财务报表记录是否差太多。

如果差异超过容忍阈值，就会在 `validation_issues` 里报出来。
